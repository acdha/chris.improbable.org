Hello from Gondor

Eldarion has been building an infrastructure to host our own and client sites. We are beginning the process of making it generally available.

Gondor has been something James Tauber and I have been talking about ever since our first Eldarion meeting at the Hyatt Summerfield Suites in Boston. At that time it only described the slices we purchased on Slicehost. We discussed ideas of how we could create tools for the infrastructure to simplify much of the deployment process. As we were stepping in the elevator to head off to dinner James suggested a command-line interface almost exactly like you see on our homepage today.

Fast-forward nearly two years and here we are. We have built out the Gondor infrastructure for many sites of our own as well as those of our clients. Now we're getting serious about making Gondor generally available.

While the original Gondor was hosted on Slicehost, we decided using Rackspace Cloud was going to be best for the infrastructure moving forward. It allows us more flexibility when it comes to servers. We've been building out what we've internally dubbed "Gondor 2.0". It currently supports:

  • revision control via git or mercurial
  • dependency management using pip
  • database migrations via South or nashvegas
  • full backups of your entire application
  • asychronous and scheduled task execution
  • full-text search using Solr and django-haystack
  • caching via memcached or redis

We are working hard to make this available to a small set of users other than ourselves. Put yourself on our waiting list for the private beta by visiting the homepage. Stay tuned to this blog as we will be putting out more details about the infrastructure and tools soon.
