Chrome Web Store: a solution in search of a problem?

So two things they are not mentioning here, one that I very weakly buy into and one I actually use for Oplop.

First, the apps have upfront permissions, so no more having to clear a website to allow a feature after you have already landed on the page (this is the Chromium team's argument). For instance, when installing an app you can declare you want infinite storage and geo-location access upfront instead of having to click OK at the top of the browser window.

Two, apps can be "hosted", meaning that they are fully downloaded and run locally instead of simply acting as a shortcut link to a website. While this might superficially seem like HTML5 app caching, it's more than that as you get access to Chrome extension APIs. This means cross-site XHR, browser details, etc. This ability does allow for more powerful HTML5-based apps than one could have by simply visiting a URL.

Google launched its Chrome Web Store yesterday alongside its announcement of the Chrome OS pilot program. Google seems to think that the modern Web, which is increasingly capable of delivering desktop-like application experiences, needs an application delivery channel, even though there is nothing to actually deliver—except URLs.

The way that users consume applications in the desktop and mobile world is fundamentally different than they way that they do it on the Web—where paywalls are often reviled and there is little distinction between content and software. In such an environment, does the application store model make any sense? We are not convinced.
