Sprint! Our first Google Refine extension: refine-stats

Background

Recently Google released their post-buyout refresh of Freebase Gridworks, now called Google Refine. If you haven’t checked it out yet: DOWNLOAD IT NOW. This is an incredible tool for journalists, analysts or developers who work with data. Once you’ve clustered and reconciled your crufty public dataset into a glistening gem of normality you won’t know how you lived without it.

The plan

Yesterday Joe and I made a spontaneous decision to spend the day sprinting to be the first ones to release a third-party Google Refine extension. Our motivations for this were several: 1) learn about the process of writing extensions, 2) prove that we could write an extension that would be useful in our day-to-day work, 3) show our support for an incredible product and 4) be awesome. We discussed a long list of things we would like to build, including:

  • Map facet
  • Postgres import
  • Postgres export
  • KML export
  • Single-click geocoding

We decided that well all of these were probably possible, we really ought to start with something simpler and thus increase our chances of seeing the project through to completion. To that end we decided to build a package that would compute basic statistics (mean, median, mode, etc) for a column of data. This the sort of task that we might often do at the same time we are cleaning up data and normally would have to drop into a different tool to accomplish (typically R).

With a project in mind, we started sprinting.

At 5:20 PM, shortly before leaving for our monthly OpenGovChicago meetup, we published working code.

Output: refine-stats

We essentially succeeded in all our stated goals. There is certainly cruft in the code and our documentation is totally weak, but the extension does work and provides the information we were looking for.
