Git Improvements

Jason Rudolph / @jasonrudolph:

As of #git 1.6.6, tracking & checking out a remote branch now works the way you always wished it had. http://is.gd/5LIV5

Nice.

I have a Yojimbo note dedicated to git incantations, and setting up tracking branches was one of them.

Perhaps it isn’t a futile wish that one day submodules will be cloned by default, or one day we’ll have a way to read the current branch’s name without resorting to sed.

I’m also happy to see the git project is willing to make behavioral changes at the cost of breaking backward compatibility.

In the grand scheme of things, git is still very young and it’s good to see the project hasn’t grown arthritic yet.

Update: Previously I used 

git branch 2> /dev/null | sed -e '/^[^*]/d' -e 's/* \(.*\)/\1/'

to extract the current branch’s name, but Nathan de Vries informed me of 

git describe --contains --all HEAD

Thanks, Nathan. That’s much better.

Update: Vincent Driessen writes::

Regarding your blog post @ http://is.gd/6vHl7 : git name-rev —name-only HEAD also does the trick and is “cleaner”, I think.
