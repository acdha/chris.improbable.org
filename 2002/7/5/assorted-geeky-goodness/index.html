Assorted geeky goodness

I recently started using eMusic and have been rounding out the more obscure parts of my music collection. I quickly ran into a problem - there aren't any decent RMP download tools for the Mac.

TafkaR strips the ID3 tags from the files it downloads and shows its OS9 roots in the way it mangles file and album names.

After deciding to write a little utility to handle this, I considered perl and XML::Simple, which I've heard good things about. Somewhere around the 20th CPAN module dependency, I remembered why I don't use Perl more.

A couple hours and 200 lines of PHP later, I have a command-line tool which parses the MP3 files, stores them in the appropriate artist/album subfolders in my music directory and adds them to a new music playlist in iTunes using osascript. Along the way, it became obvious that eMusic's programmers don't believe in well-formed XML and many of their MP3 files are incompletely tagged. Sure, I have AppleScript to give iTunes the correct track information but . . . wouldn't it have been easier for them to have made their ID3 tags match their database in the first place?

At some point I plan on rewriting this as a proper GUI program using Omnigroup's networking frameworks so I can show a proper progress list, give the user more control over how files are handled and give the option of setting the file association for .rmp files appropriately.
