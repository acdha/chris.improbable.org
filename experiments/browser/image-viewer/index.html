Experiment in producing a pure CSS fit/centered image viewer

Experiment using CSS display:table and viewport units to produce a vertically and horizontally centered image which is resized to fit within the window. display:table is necessary to vertically center the image when it is smaller than the window width/height; viewport units are necessary because browsers will expand the document to contain the contents even if you specify max-width: 100% from <html> down to the element in question

Browser Bugs

  1. WebKit: Safari and Chrome only calculate units on the initial page load: WebKit #114690 Chromium #231963
  2. Safari on iOS6: the viewport units are either ignored or miscalculated to expand to contain the largest element which was ever displayed - i.e. if JavaScript is used to toggle the image constraints the page will forever be larger. Safari on Mac OS X behaves as expected.
